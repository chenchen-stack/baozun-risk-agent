# 宝尊风控 Agent — 阿里云部署（ACR + ECS）

适用于把 `agent-app` 以 Docker 方式跑在阿里云上，对外提供 `https://你的域名` 或 `http://ECS公网IP:8800`。

## 零、已准备好的脚本（推荐）

1. 在 [ACR 控制台](https://cr.console.aliyun.com/) 创建**命名空间**和镜像仓库 `baozun-risk-agent`（可改名，与 `env.deploy` 一致即可）。
2. 在 `agent-app` 下复制 `env.deploy.example` → **`env.deploy`**，填好 `ALIYUN_ACR_*`（`env.deploy` 已在 `.gitignore`，勿提交）。
3. **本机**安装并启动 **Docker Desktop**，双击 **`一键推送镜像到阿里云.bat`**（或 PowerShell：`.\scripts\deploy-from-env.ps1`）。
4. 把 **`scripts/ecs-run.sh`** 上传到 ECS，`chmod +x` 后编辑脚本内镜像地址与密钥，执行：`sudo bash ecs-run.sh`（或在 ECS 上按脚本末尾打印的手动命令操作）。

---

## 一、你需要开通/准备

| 资源 | 用途 |
|------|------|
| **容器镜像服务 ACR** | 存放镜像（个人版实例即可起步） |
| **云服务器 ECS** | 运行容器（建议 2 核 4G、Alibaba Cloud Linux 3 / Ubuntu 22.04） |
| **安全组** | 入方向放行 **8800**（若前面有 SLB 则按 SLB 规则放行 80/443） |

环境变量（在控制台或启动命令里配置，**不要写进镜像**）：

- **`DEEPSEEK_API_KEY`**：必填，与当前 `agent.py` 中 DeepSeek 接口一致。
- **`AGENT_MAX_ITERATIONS`**（可选）：单条用户消息内 Agent 与工具的循环轮次上限。代码与 compose 默认 **36**（落在推荐的 **32～40** 区间）；若仍偶发「触顶」，可改为 **40** 或更高（环境变量最高 **80**）。**修改后需重启容器 / 进程**（`server.py` 或 uvicorn）后生效。
- **`LLM_API_KEY`** / **`DEEPSEEK_API_KEY`**：大模型密钥（至少其一；亦支持前端请求头传入）。
- **`LLM_OPENAI_API_BASE`**、**`LLM_MODEL`**（可选）：对接**火山引擎 / 千问等 OpenAI 兼容网关**时使用；不填则默认 DeepSeek 官方端点 + `deepseek-chat`。
- **`FEISHU_WEBHOOK_URL`** / **`OA_WEBHOOK_URL`**（可选）：配置后，Agent 工具「飞书通知」会向真实群机器人或 OA 回调 **POST**（未配置则仍为演示写入内存列表）。
- **`ENABLE_SCHEDULED_MONTHLY_REPORT=1`**（可选）：**每月 1 号**自动生成内控 HTML 报告到 `reports/`（可与飞书 Webhook 联用推送提示）。
- **`EXTERNAL_DATA_BOOTSTRAP_URL`** 或将 JSON 放入镜像内 **`external_data/`** 目录（可选）：启动时合并进内存数据层，作为真 API 接入前的过渡（详见仓库内 `integrations/data_overlay.py`）。

健康检查旁可查看对接摘要：`GET /api/integrations/status`。

可选：**弹性公网 IP**、**负载均衡 SLB**、**SSL 证书**（HTTPS）。

## 二、构建镜像并推到 ACR

在**本机**（已安装 Docker）进入 `agent-app` 目录。

1. 在 [ACR 控制台](https://cr.console.aliyun.com/) 创建**命名空间**和**镜像仓库**（例如仓库名 `baozun-risk-agent`），记下**公网地址**，形如：  
   `registry.cn-hangzhou.aliyuncs.com/<命名空间>/baozun-risk-agent`

2. 登录（用户名一般为阿里云账号全名；密码在 ACR「访问凭证」里设置）：

```bash
docker login registry.cn-hangzhou.aliyuncs.com
```

3. 构建并推送（把 `<你的ACR地址>` 换成上一步地址，`v1` 为标签可改）：

```bash
docker build -t <你的ACR地址>:v1 .
docker push <你的ACR地址>:v1
```

---

## 三、在 ECS 上运行

1. ECS 安装 Docker（Alibaba Cloud Linux 3 示例）：

```bash
sudo yum install -y docker
sudo systemctl start docker && sudo systemctl enable docker
```

2. 登录 ACR（同本机 `docker login`）。

3. 拉取并启动（**务必替换**镜像地址和 API Key）：

```bash
sudo docker pull <你的ACR地址>:v1

sudo docker run -d --name baozun-agent --restart unless-stopped \
  -p 8800:8800 \
  -e PORT=8800 \
  -e DEEPSEEK_API_KEY="你的DeepSeek密钥" \
  -e AGENT_MAX_ITERATIONS=36 \
  <你的ACR地址>:v1
```

（可选）若已申请飞书群机器人 Webhook，可追加 `-e FEISHU_WEBHOOK_URL="https://open.feishu.cn/..."`；需要每月 1 号自动生成月报时追加 `-e ENABLE_SCHEDULED_MONTHLY_REPORT=1`。

```bash
# 示例：带飞书 + 定时月报
sudo docker run -d --name baozun-agent --restart unless-stopped \
  -p 8800:8800 \
  -e PORT=8800 \
  -e DEEPSEEK_API_KEY="你的DeepSeek密钥" \
  -e FEISHU_WEBHOOK_URL="https://open.feishu.cn/open-apis/bot/v2/hook/xxxx" \
  -e ENABLE_SCHEDULED_MONTHLY_REPORT=1 \
  <你的ACR地址>:v1
```

4. 浏览器访问：`http://<ECS公网IP>:8800/`  
   健康检查：`http://<ECS公网IP>:8800/api/health`

5. 更新版本：重新 `docker build` / `push` 新标签，在 ECS 上 `docker stop/rm` 后按上面 `docker run` 换为新标签。

## 四、使用 docker-compose（可选）

本目录下 `docker-compose.aliyun.yml` 可从 ACR 拉镜像并启动。在 ECS 上：

```bash
export ACR_IMAGE=registry.cn-hangzhou.aliyuncs.com/命名空间/baozun-risk-agent:v1
export DEEPSEEK_API_KEY=你的密钥
# 可选：复杂任务可调大，例如 export AGENT_MAX_ITERATIONS=40
docker compose -f docker-compose.aliyun.yml up -d
```

## 五、Serverless 应用引擎 SAE（免运维）

适合不想管 ECS 的场景：

1. 将镜像推送到 ACR（同上）。  
2. 打开 [SAE 控制台](https://sae.console.aliyun.com/) → 创建应用 → 选择**镜像部署**。  
3. 镜像选 ACR 中的地址；**容器端口**填 **8800**（与 `Dockerfile` / `fly.toml` 一致）。  
4. 在应用环境变量中配置 **`DEEPSEEK_API_KEY`**；可选配置 **`AGENT_MAX_ITERATIONS`**（默认 36，复杂任务可设 32～40）。若 SAE 注入 **`PORT`**，本镜像已支持，与控制台端口配置保持一致即可。  
5. 绑定 SAE 提供的访问域名或自定义域名、HTTPS。

具体字段以控制台当前版本为准。

---

## 六、说明

- **`reports/`** 目录在容器内为临时存储，**重建实例后报告文件会丢失**；若要长期保存，需挂载 NAS/OSS 等（需改应用或挂载卷，属后续优化）。  
- 生产环境建议：**HTTPS**、**仅内网访问 ACR**、**密钥用 KMS/环境变量**，并限制安全组来源 IP。
