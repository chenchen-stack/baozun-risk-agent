# 非经营性采购全流程与风控控制点

> 以"采购一批办公桌椅"为例，按集团企业真实执行标准，拆解从需求产生到流程彻底结束的**每一步**，以及风控部门在每一步介入的角度。

---

## 全景视图

一笔非经营性采购的完整生命周期包含 **10 个阶段、30+ 个控制点**：

```
❶ 预算编制与额度管控
   ↓
❷ 需求提出与审批
   ↓
❸ 供应商寻源与比价
   ↓
❹ 合同签订与审批
   ↓
❺ 下单与订单跟踪
   ↓
❻ 到货验收与入库
   ↓
❼ 发票接收与校验
   ↓
❽ 三单匹配与应付挂账
   ↓
❾ 付款审批与执行
   ↓
❿ 资产入账与后续管理（固定资产才有）
```

下面逐一展开。

---

## ❶ 预算编制与额度管控

### 业务动作

每年年底，各部门编制下一年度的非经营性采购预算，经财务部汇总、管理层审批后下达年度预算额度。预算通常按费用科目（办公用品、IT设备、装修工程、市场物料等）和部门两个维度切分。

### 具体细节

| 步骤 | 动作 | 产出物 |
|------|------|--------|
| 1 | 各部门根据历史用量和来年计划，编制部门年度采购预算 | 《部门年度采购预算表》 |
| 2 | 财务部汇总各部门预算，编制集团层面的总预算 | 《集团年度非经营性采购预算总表》 |
| 3 | 管理层（CFO/总经理/预算委员会）审批 | 批准后的预算下达文件 |
| 4 | 财务系统录入各部门/科目的预算额度 | 系统中的预算池 |

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **预算合理性** | 对比历史实际支出，偏差过大的要求说明原因（为什么今年比去年多30%？是扩张了还是虚报了？） |
| **预算完整性** | 是否有费用科目遗漏，避免后续走"超预算特批"绕过管控 |
| **预算颗粒度** | 预算是否细化到可执行的层面（只写"办公费100万"太粗，应拆到"家具20万、耗材30万、IT设备50万"） |

## ❷ 需求提出与审批

### 业务动作

业务部门实际要采购时，先在OA/采购系统中提交采购申请单。

### 具体细节

| 步骤 | 动作 | 产出物 |
|------|------|--------|
| 1 | 需求人填写《采购申请单》：品名、规格、数量、预估单价、总金额、用途说明、期望到货日期 | 采购申请单 |
| 2 | 部门负责人审批：核实是否真的需要、数量是否合理 | 部门负责人签批 |
| 3 | 行政/采购部审核：核对库存（仓库有没有现货可调配）、确认规格标准（公司统一标准是什么型号的桌椅） | 行政确认意见 |
| 4 | 财务部审核：检查是否在预算额度内、费用科目是否正确 | 财务审核意见 |
| 5 | 分级审批：按金额走不同层级审批 | 最终审批签字 |

### 分级审批标准（典型集团标准）

| 金额 | 审批权限 |
|------|---------|
| ≤5,000元 | 部门负责人 + 行政部 |
| 5,000-50,000元 | 部门负责人 + 行政部 + 财务部 |
| 50,000-200,000元 | 以上 + 分管副总裁 |
| 200,000-500,000元 | 以上 + CFO |
| >500,000元 | 以上 + 总裁/总经理（部分企业需上董事会） |

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **需求真实性** | 这个部门真的缺桌椅吗？上次采购了多少？用了多少？是否存在虚报需求套取采购资金？ |
| **预算占用** | 这笔采购审批后，该部门/科目的年度预算还剩多少？是否会导致后续月份无预算可用？ |
| **审批权限合规** | 金额是否匹配了正确的审批层级？有无越级审批、审批人回避（自己批自己的采购） |
| **拆单规避** | 是否有人故意把一笔10万的采购拆成两笔4.9万的来规避更高层级审批？（看同一申请人/同一供应商/同一品类短期多次小额申请） |
| **重复采购** | 跟历史申请对比，同部门短期内是否重复采购同类物资？ |

## ❸ 供应商寻源与比价

### 业务动作

采购申请审批通过后，采购部负责寻找供应商、询价比价、确定中标供应商。

### 具体细节

| 金额区间 | 寻源方式 | 具体要求 |
|----------|---------|---------|
| ≤10,000元 | 直接采购 | 至少1家报价，可从合格供应商库直接选 |
| 10,000-100,000元 | 比价采购 | 至少**3家**供应商书面报价，编制《比价分析表》 |
| 100,000-500,000元 | 竞争性谈判/邀请招标 | 至少3-5家参与，组建评审小组（≥3人），纪检/风控列席 |
| >500,000元 | 公开招标 | 发布招标公告，资格预审，开标/评标/定标全程留痕 |

**比价分析表内容**：
- 各供应商名称、报价、交货期、质保期、付款条件
- 历史合作记录和评价
- 推荐理由（不能只选最低价，需综合评定）

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **围标串标** | 参与报价的3家供应商之间是否存在关联关系？（同一法人、同一注册地址、交叉持股、投标文件雷同） |
| **回扣利益输送** | 是否总是选同一家供应商？历史中标率异常高的供应商需标记。采购员与供应商之间是否有异常往来 |
| **价格合理性** | 中标价与市场价对比偏离度如何？与历史同类采购价格对比是否异常？（某企业案例：高于市场价30%采购，经办人收取3%-10%回扣，涉案超1亿元） |
| **虚假报价** | 参与比价的供应商是否是真实的？有没有"陪标"——其他家只是来凑数的？ |
| **供应商资质** | 中标供应商的工商注册、经营范围、资质证书是否齐全且有效？有无行政处罚、失信记录？ |
| **规避招标** | 本应走招标流程的金额，是否被拆分成多笔小金额的比价采购来规避？ |

**真实案例参考**：
- 某企业3名员工与20家供应商勾结，高价采购收回扣，涉案超1亿元（2025年江西赣州案）
- 某采购主管以"借款"名义收回扣10年、非法获利242万元（2025年河南案）
- 某高校中标85万元防火墙设备实际是几百元路由器（虚假材料投标）

## ❹ 合同签订与审批

### 业务动作

确定供应商后，双方签订采购合同。合同需经过内部法务、财务、风控等部门会签。

### 具体细节

| 步骤 | 动作 | 产出物 |
|------|------|--------|
| 1 | 采购部起草合同（或使用标准模板），明确：品名、规格、数量、单价、总价、交货时间、交货地点、验收标准、质保期、付款方式、违约责任 | 合同初稿 |
| 2 | 法务部审核：条款合规性、违约责任是否对等、知识产权、保密条款 | 法务审核意见 |
| 3 | 财务部审核：付款条件、税率、发票要求、费用科目 | 财务审核意见 |
| 4 | 风控部审核（部分企业）：合同风险评估 | 风控审核意见 |
| 5 | 分管领导/授权代表签署 | 签署完毕的合同 |
| 6 | 合同归档（合同管理系统 + 纸质归档） | 合同编号，入系统 |

### 合同关键条款（风控必查项）

| 条款 | 风控检查要点 |
|------|-------------|
| **标的物条款** | 品名、规格、数量是否与采购申请单一致？不能出现"桌椅若干"这种模糊表述 |
| **价格条款** | 单价和总价是否与中标价一致？是否含税？税率是多少？ |
| **付款条款** | 预付比例是否合理？（一般办公家具不超过30%预付）账期多长？收款账户是否是供应商对公账户？ |
| **验收条款** | 验收标准是否明确？谁来验收？验收不合格的处理方式？ |
| **违约条款** | 逾期交货的罚则？质量不合格的赔偿？是否对等（不能只约束供应商不约束自己，也不能反过来） |
| **签署主体** | 合同签署方是否与报价方、供应商库中的主体一致？有无"张冠李戴" |

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **合同与申请一致性** | 合同标的（品名/数量/金额）是否与审批通过的采购申请单一致？有无"偷梁换柱"——申请批了A但合同签了B |
| **付款条件风险** | 预付比例是否过高？（全额预付是高风险信号）有无资金回收保障？ |
| **关联交易** | 签约供应商与本公司人员是否存在关联关系？（亲属开的公司、前员工创办的公司等） |
| **合同条件恶化** | 与该供应商历史合同对比，本次条件是否变差？（账期变短、价格变高、违约金降低） |
| **阴阳合同** | 系统中的电子合同与实际执行的纸质合同是否一致？ |

## ❺ 下单与订单跟踪

### 业务动作

合同签署后，采购部向供应商正式下达采购订单（PO），并跟踪交货进度。

### 具体细节

| 步骤 | 动作 | 产出物 |
|------|------|--------|
| 1 | 在采购系统中创建采购订单（PO），关联合同编号 | 采购订单（PO号） |
| 2 | 将PO发送给供应商，供应商确认 | 订单确认书 |
| 3 | 跟踪生产/备货进度 | 进度记录 |
| 4 | 确认发货、物流跟踪 | 物流信息 |

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **PO与合同一致性** | 下单数量、单价、规格是否与合同完全一致？ |
| **重复下单** | 同一合同是否被重复创建了多个PO？ |
| **交货延迟** | 超过合同约定交货期的，是否触发违约条款？是否有人私下放宽了交期？ |

## ❻ 到货验收与入库

### 业务动作

货物到达后，由仓库/行政部联合需求部门共同验收，确认数量和质量。

### 具体细节

| 步骤 | 动作 | 产出物 |
|------|------|--------|
| 1 | 货物到达，仓库/行政部签收，核对送货单与PO | 签收记录 |
| 2 | **数量验收**：逐项清点，与PO数量核对 | 数量核对表 |
| 3 | **质量验收**：检查规格型号是否与合同约定一致、外观有无损坏、功能是否正常（如办公椅的升降、轮子） | 质量验收记录 |
| 4 | 需求部门代表参与验收确认 | 需求部门签字 |
| 5 | 验收合格 → 编制《验收入库单》（GRN），录入系统 | 入库单（GRN号） |
| 6 | 验收不合格 → 填写《质量异常报告》，启动退货/换货/补货流程 | 异常报告 |

**验收的"不相容"原则**：采购人员不能同时担任验收人员——"买的人不能自己验自己买的东西"。

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **到货与PO一致性** | 实际到货的品名、规格、数量是否与PO一致？有无"以次充好"——合同约定真皮椅，到货变成仿皮椅？ |
| **验收人独立性** | 验收人是否与采购经办人不同？有无"自买自验"？ |
| **验收时效** | 从到货到完成验收是否在合理时间内（一般3-5个工作日）？拖延验收可能是在等供应商"补货"或掩盖短少 |
| **部分到货** | 合同约定100把椅子，只到了80把，是否正确记录了部分到货？后续20把的交货计划？ |
| **虚假验收** | 有没有货没到但验收单已经签了的情况？（配合供应商提前开票、提前付款） |

## ❼ 发票接收与校验

### 业务动作

供应商根据实际交货情况开具发票，财务部接收并校验。

### 具体细节

| 步骤 | 动作 | 产出物 |
|------|------|--------|
| 1 | 供应商开具增值税专用发票（或普通发票），寄送至财务部 | 发票原件 |
| 2 | 财务部登记发票台账：发票代码、号码、金额、税额、开票日期 | 发票台账 |
| 3 | **发票验真**：通过税务局接口验证发票真伪 | 验真结果 |
| 4 | **发票内容校验**：品名、规格、数量、单价、税率是否与PO/合同一致 | 校验记录 |
| 5 | **发票合规性检查**：开票方名称是否与合同签约方一致、纳税人识别号是否正确、发票是否在有效期内 | 合规检查记录 |

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **虚开发票** | 开票方与实际供货方是否一致？有无第三方代开发票？（四流一致的核心检查点） |
| **发票连号** | 同一供应商短期内开出的多张发票是否连号？（连号可能意味着虚开） |
| **大头小尾** | 发票金额是否与实际交易金额一致？有无多开/少开？ |
| **过期发票** | 发票开具日期是否与交易时间匹配？有无"补开"很久以前的发票？ |
| **重复发票** | 发票代码+号码在系统中是否唯一？同一张发票是否被提交了两次？ |
| **税率正确性** | 办公家具一般适用13%增值税税率，发票上的税率是否正确？ |

## ❽ 三单匹配与应付挂账

### 业务动作

这是整个P2P流程的**核心控制点**——财务部将采购订单（PO）、入库单（GRN）、发票三者进行匹配校验，匹配通过后在财务系统生成应付账款。

### 三单匹配的具体规则

```
      采购订单（PO）          入库单（GRN）          发票
      ┌───────────┐         ┌───────────┐       ┌───────────┐
      │ 品名: 办公椅│         │ 品名: 办公椅│       │ 品名: 办公椅│
      │ 数量: 100把 │   ←→   │ 数量: 100把 │  ←→  │ 数量: 100把 │
      │ 单价: 800元 │         │             │       │ 单价: 800元 │
      │ 税率: 13%   │         │             │       │ 税率: 13%   │
      │ 总价: 90,400│         │             │       │ 总价: 90,400│
      └───────────┘         └───────────┘       └───────────┘
```

**匹配容差标准**（典型集团标准）：

| 匹配项 | 容差 | 超差处理 |
|--------|------|---------|
| 数量 | PO数量与GRN数量差异 ≤ **2%** | 冻结，人工复核 |
| 单价 | PO单价与发票单价差异 ≤ **1%** | 冻结，人工复核 |
| 税率 | 必须 **100%一致** | 直接退回 |
| 金额 | 三者计算结果差异 ≤ 一定阈值（如50元） | 自动调整或人工复核 |
| 供应商 | PO供应商 = GRN供应商 = 发票开票方 | 必须一致，不一致直接退回 |

**匹配状态**：
- ✅ **完全匹配** → 自动生成应付凭证
- ⚠️ **部分匹配**（如部分到货、部分开票）→ 挂起，等待后续到货/发票后继续匹配
- ❌ **匹配失败** → 生成异常工单，按异常类型分配责任人

**异常类型及闭环时限**：

| 异常类型 | 说明 | 闭环时限 |
|----------|------|---------|
| 数量不符 | 入库数量与PO不一致 | T+3天 |
| 单价不符 | 发票单价与PO约定不一致 | T+5天 |
| 税率不符 | 发票税率与PO约定不一致 | T+3天 |
| 缺票 | 已入库但供应商未开票 | T+10天 |
| 错票 | 发票信息错误需重开 | T+10天 |
| 红冲重开 | 发票需作废红冲后重新开具 | T+30天 |

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **匹配异常率** | 本月异常率是多少？是否比上月恶化？哪些供应商的异常率最高？ |
| **异常闭环率** | 产生的异常工单是否在规定时限内关闭？长期挂起的异常是什么原因？ |
| **无PO付款** | 有无绕过PO直接挂账付款的情况？（高风险信号：说明有采购绕过了前置审批） |
| **暂估入账** | 货到票未到的暂估挂账是否在合理期限内取得发票核销？长期暂估是什么原因？ |
| **四流一致** | 合同签约方 = PO供应商 = 入库送货方 = 发票开票方 = 收款账户方——五者必须一致 |

## ❾ 付款审批与执行

### 业务动作

三单匹配通过、应付挂账完成后，财务部按合同约定的付款条件发起付款申请。

### 具体细节

| 步骤 | 动作 | 产出物 |
|------|------|--------|
| 1 | 财务部编制《付款申请单》，附三单匹配结果、合同复印件 | 付款申请单 |
| 2 | **付款前复核**：检查合同条款是否已满足付款条件（如"验收合格后30天内付款"，验收是否已完成？30天到了吗？） | 复核记录 |
| 3 | **分级审批**（与采购申请的分级逻辑类似，但更严格） | 审批记录 |
| 4 | **资金部/出纳执行付款**：通过银企直连系统批量付款 | 银行回单 |
| 5 | **核销**：将付款记录与应付账款进行核销 | 核销记录 |
| 6 | **对账**：定期与供应商对账确认 | 对账单 |

### 付款前的"最后一道关"检查清单

| 检查项 | 说明 |
|--------|------|
| 三单匹配是否通过？ | PO + GRN + 发票 三者一致 |
| 合同付款条件是否满足？ | 如"验收后30天"，日期是否到了？ |
| 收款账户是否与合同一致？ | 不一致需重新走合同变更审批 |
| 是否有扣款项？ | 质量罚款、逾期交货罚款是否已扣除？ |
| 是否重复付款？ | 同一发票/同一PO是否已经付过款了？ |
| 预付款是否已抵扣？ | 如已预付30%，本次应付70% |
| 税款是否正确？ | 增值税进项税额是否正确抵扣？ |

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **重复付款** | 跨系统比对，同一供应商+同一金额+同一时间段的付款是否重复？ |
| **大额异常** | 单笔付款金额是否超出合同总额？是否远超该供应商历史平均付款金额？ |
| **异常时间** | 有无非工作时间（节假日、凌晨）的付款？有无月末/季末突击付款？ |
| **收款账户变更** | 供应商是否临时变更收款账户？（高风险信号——可能是诈骗或资金转移） |
| **审批完整性** | 付款审批链是否完整？有无跳过某一审批环节？ |
| **先付后审** | 有无先付款后补审批的情况？ |
| **资金回流** | 付款后是否有资金从供应商账户回流至公司内部人员账户？（严禁） |

## ❿ 资产入账与后续管理

> 如果采购标的是桌椅这类满足固定资产标准（一般单价≥2000元或5000元，使用年限≥1年）的物资，还需要进入资产管理流程。

### 具体细节

| 步骤 | 动作 | 产出物 |
|------|------|--------|
| 1 | **资产入账**：按不含税价录入固定资产模块，生成资产卡片（资产编号、名称、规格、数量、原值、购入日期、使用部门、责任人） | 固定资产卡片 |
| 2 | **实物标签**：打印资产标签贴于实物上（条形码/二维码） | 贴标完成 |
| 3 | **折旧计提**：当月增加次月开始计提折旧。办公家具按5年直线法，残值率5%。月折旧额 =（原值 × 95%）÷ 60 | 月度折旧分录 |
| 4 | **定期盘点**：每年至少一次全面盘点，季度重点盘点高价值资产 | 盘点报告 |
| 5 | **日常管理**：调拨（部门间移交）、维修、保养 | 调拨单、维修记录 |
| 6 | **报废处置**：达到使用年限或无法修复时 → 技术鉴定 → 报废申请 → 审批 → 残值回收 → 账务清理 | 报废审批单、残值回收单 |

**"账、卡、物"三者必须100%一致**：
- **账**：财务系统中的固定资产账面记录
- **卡**：资产卡片/台账中的登记信息
- **物**：实物的实际存在、位置、状态

### 风控部门关注什么

| 风控角度 | 具体检查 |
|----------|---------|
| **账实不符** | 盘点时发现实物少了（盘亏）——是丢失了还是被私自处置了？实物多了（盘盈）——是否有未入账的采购？ |
| **虚假报废** | 设备还能用但申请报废，残值被低价处置给关联方？ |
| **私自处置** | 有无未经审批直接变卖或赠送资产的情况？ |
| **折旧准确性** | 折旧计提是否按规定执行？有无随意变更折旧年限操纵利润？ |
| **责任人变更** | 人员离职/调岗时，名下资产是否办理了移交手续？ |

## 风控部门的系统性视角：不是查一个环节，而是看全局

上面每个环节的风控关注点是"纵向"的逐环检查。但一个成熟的风控部门还会做**"横向"的系统性审视**：

### 横向审视一：不相容职务分离

整条链路上，以下角色**必须由不同的人担任**：

| 不相容职务对 | 说明 |
|-------------|------|
| 需求提出 ↔ 需求审批 | 不能自己提、自己批 |
| 询价比价 ↔ 确定供应商 | 不能自己比、自己定 |
| 合同起草 ↔ 合同审批 | 不能自己写、自己签 |
| 采购执行 ↔ 验收确认 | 不能自己买、自己验 |
| 验收入库 ↔ 会计记录 | 不能自己收、自己记账 |
| 付款审批 ↔ 付款执行 | 不能自己批、自己付 |

**风控检查方法**：拉出每笔采购的全流程参与人清单，检查是否有同一个人在上述不相容角色中出现了两次。

### 横向审视二：供应商集中度与依赖性

| 指标 | 风险阈值 | 说明 |
|------|---------|------|
| 单一供应商占采购总额比例 | >30% 预警 | 对单一供应商的过度依赖 |
| 前5大供应商集中度 | >70% 预警 | 供应商结构不健康 |
| 新供应商占比 | >50% 关注 | 为什么突然引入大量新供应商？ |
| 供应商存续年限 | <1年 关注 | 刚成立的公司就拿到大合同？ |

### 横向审视三：时间维度的异常模式

| 模式 | 说明 |
|------|------|
| **年末突击采购** | 12月采购量占全年比例异常高——可能是花预算、调利润 |
| **月末集中付款** | 月末最后3天付款量占全月>40%——可能是赶账期或突击出账 |
| **审批秒过** | 审批流程从发起到完成<5分钟——审批人可能没有实质审核 |
| **闭环超时** | 异常工单长期不关闭——可能存在故意拖延或放水 |

### 横向审视四：数据贯通性检查（最核心的Agent价值点）

```
采购申请单 ←→ 合同 ←→ PO ←→ GRN ←→ 发票 ←→ 付款记录 ←→ 资产卡片

   ↑           ↑       ↑       ↑       ↑         ↑           ↑
  OA系统    合同系统  采购系统  仓库系统  财务系统   银行系统    资产系统

                    七个系统，一条链路
               每个系统有自己的编号体系
          人工要把这条链路从头到尾串起来 = 噩梦
              Agent 10秒完成 = 价值
```

这就是宝尊的核心痛点，也是风控智能体的核心价值所在。
